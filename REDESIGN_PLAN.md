## 4. Визуальные изменения

### 4.1 Типографика

**Заголовки:**
- `h1`: `clamp(3rem, 8vw, 6rem)` — очень крупный, как в референсе
- `h2`: `clamp(2rem, 5vw, 3.5rem)` — крупные секционные заголовки
- `h3`: `clamp(1.5rem, 3vw, 2rem)` — подзаголовки

**Текст:**
- Базовый размер: `1rem` (16px)
- Межстрочный интервал: `1.75` (больше воздуха)
- Letter-spacing: `-0.02em` для заголовков

**Variable Font:**
- Использовать Inter Variable или Manrope Variable
- Веса: 400 (body), 500 (subheadings), 700 (headings), 800 (hero)

### 4.2 Цветовая палитра

**Обновление существующих цветов:**
```css
/* Основные цвета (адаптировать под дизайн-систему) */
--base-color-background: #fafafa; /* Light */
--base-color-foreground: #0a0a0a; /* Dark text */

/* Акцентные цвета из логотипа */
--logo-color-azure: #99b9ff;
--logo-color-springgreen: #78ffd1;
--logo-color-flamingo: #ffb3c2;
--logo-color-lime: #f0ffa6;

/* Использование: */
--base-color-accent: var(--logo-color-azure); /* Основной акцент */
```

### 4.3 Spacing (отступы)

**Увеличить отступы для воздуха:**
```css
--spacing-section: clamp(6rem, 12vw, 10rem); /* Больше между секциями */
--spacing-grid: clamp(2rem, 4vw, 3rem); /* Больше между элементами */
```

### 4.4 Декоративные элементы

**Модульные формы на фоне:**
- Hero: DECONSTRUCTED, BLURRED_COLOUR, opacity: 0.1-0.2
- About: S или R, OUTLINE, opacity: 0.3
- Projects: небольшие формы между карточками

**Позиционирование:**
- Абсолютное позиционирование
- z-index: -1 (за контентом)
- Размытие через `filter: blur(40px)`

## 5. Анимации и интерактивность

### 5.1 Появление элементов

**Scroll animations:**
- Элементы появляются при скролле (fade-in + slide-up)
- Использовать Intersection Observer API
- Или CSS `@keyframes` + `animation-delay`

### 5.2 Hover-эффекты

**Карточки проектов:**
- Масштабирование: `transform: scale(1.02)`
- Смена цвета границы
- Плавный переход: `transition: all 300ms ease`

**Ссылки:**
- Подчёркивание снизу при hover
- Смена цвета на акцентный

### 5.3 Интерактивный логотип

**В header:**
- Hover на "Sayan" → SR
- Hover на "Roor" → RS
- Плавный морфинг форм (если реализовано)

## 7. Технические детали

### 7.1 Новые зависимости (опционально)

**Для анимаций:**
- `framer-motion` — для сложных анимаций (scroll, stagger)
- `react-intersection-observer` — для scroll animations

**Рекомендация:** Начать с CSS-анимаций, добавить библиотеки только если нужны сложные эффекты.

### 7.2 Оптимизация изображений

**Проекты:**
- Использовать `next/image` с `priority={false}`
- Формат: WebP/AVIF
- Lazy loading для изображений ниже fold

### 7.3 Производительность

**Меры:**
- Ленивая загрузка анимаций (только при видимости)
- CSS-анимации вместо JS где возможно
- `will-change` для анимируемых элементов
- Минификация SVG

### 7.4 Accessibility

**Требования:**
- Все интерактивные элементы с `aria-label`
- Поддержка `prefers-reduced-motion`
- Достаточный контраст цветов (WCAG AA)
- Клавиатурная навигация

## 10. Риски и митигация

### Риск 1: Перегрузка анимациями
**Митигация:** Начать с простых CSS-переходов, добавлять сложные только при необходимости.

### Риск 2: Потеря функциональности при упрощении
**Митигация:** Сохранить все ключевые функции (бриф, навигация), только изменить визуальное представление.

### Риск 3: Сложность реализации Logo компонента
**Митигация:** Начать с простых SVG-форм, итеративно усложнять.

### Риск 4: Производительность на мобильных
**Митигация:** Оптимизировать изображения, ленивая загрузка, отключить сложные анимации на мобильных.
